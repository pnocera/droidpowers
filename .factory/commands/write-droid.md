# Writing Droids Command

## Usage
```bash
/droid writing-skills
```

## Description
Applies TDD methodology to create and validate new droids through pressure testing.

## Process
1. RED phase: Create pressure scenarios and document baseline failures
2. GREEN phase: Write minimal droid addressing specific violations  
3. REFACTOR phase: Close loopholes until bulletproof against rationalization
4. Deploy: Validate YAML frontmatter and Factory AI compliance

## Examples
```bash
/droid writing-skills
# Guides through TDD process for creating robust droids
```
